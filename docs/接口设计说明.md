# 租房微信小程序接口设计说明

## 1. 文档目标

- 定义前端 `services` 层与云函数之间的统一调用规范
- 明确当前版本每个 service 对外暴露的方法、参数和返回结构
- 统一 token 鉴权、错误码和受保护接口口径

## 2. 调用规范

### 2.1 总原则

- 页面禁止直接调用 `wx.cloud.callFunction`
- 页面只能调用 `services/*.service.js`
- 所有云函数调用统一经由 `services/cloud/call.js`
- 所有异步逻辑统一使用 `async/await`

### 2.2 统一请求格式

```json
{
  "action": "getCurrentUser",
  "payload": {},
  "auth": {
    "accessToken": "client_token"
  }
}
```

说明：

- `auth` 仅在本地存在 token 时自动附带
- 公共接口可以不依赖 `auth`
- 受保护接口要求 `auth.accessToken` 有效

### 2.3 统一响应格式

成功：

```json
{
  "code": 0,
  "data": {},
  "message": ""
}
```

失败：

```json
{
  "code": 401,
  "data": null,
  "message": "未登录或登录已过期"
}
```

### 2.4 登录成功返回结构

适用于：

- `wechatLogin`
- `loginWithPhoneCode`
- `loginWithPassword`
- `register`

返回结构：

```json
{
  "userInfo": {
    "userId": "user_1001",
    "nickName": "张三",
    "avatarUrl": "",
    "role": "tenant",
    "phone": "13800000000",
    "verified": false,
    "identityStatus": "unsubmitted",
    "wechatId": "",
    "province": "",
    "city": "",
    "district": "",
    "idCardMasked": "",
    "wechatBound": false
  },
  "accessToken": "client_token",
  "expiresAt": "2026-04-14T10:00:00.000Z"
}
```

## 3. 前端服务层接口

### 3.1 `auth.service.js`

#### `wechatLogin(userInfo)`

- 用途：微信授权登录并同步基础资料
- 返回：`{ userInfo, accessToken, expiresAt }`

#### `sendSmsCode(phone)`

- 用途：发送短信验证码
- 参数：

```json
{
  "phone": "13800000000"
}
```

#### `verifySmsCode(phone, code)`

- 用途：校验短信验证码

#### `loginWithPhoneCode(phone, code)`

- 用途：手机号验证码登录
- 返回：`{ userInfo, accessToken, expiresAt }`

#### `loginWithPassword(phone, password)`

- 用途：手机号密码登录
- 返回：`{ userInfo, accessToken, expiresAt }`

#### `register(formData)`

- 用途：注册新账号
- 返回：`{ userInfo, accessToken, expiresAt }`
- 参数：

```json
{
  "nickName": "张三",
  "phone": "13800000000",
  "role": "tenant",
  "wechatId": "wechat_01",
  "password": "123456"
}
```

#### `resetPassword(phone, code, newPassword)`

- 用途：短信验证码重置密码
- 返回：`{ reset, revokedSessions }`

#### `bindWechat()`

- 用途：将当前登录账号与当前微信身份绑定
- 返回：`{ userInfo }`

#### `unbindWechat()`

- 用途：解绑当前账号上的微信身份
- 返回：`{ userInfo }`

#### `logout()`

- 用途：注销当前登录 session
- 返回：`{ revoked: true | false }`

#### `verifyIdentity(realName, idCard)`

- 用途：提交身份资料，进入待审核状态
- 返回：`{ userInfo }`

### 3.2 `user.service.js`

#### `getCurrentUser()`

- 用途：获取当前登录用户
- 要求：受保护接口，需要有效 `accessToken`

#### `updateProfile(updateData)`

- 用途：更新个人资料
- 要求：受保护接口
- 限制：不支持通过该接口直接修改手机号
- 参数：

```json
{
  "nickName": "李四",
  "wechatId": "lisi_01",
  "province": "Guangdong",
  "city": "Shenzhen",
  "district": "Nanshan",
  "avatarUrl": ""
}
```

#### `changePassword(oldPassword, newPassword)`

- 用途：修改密码
- 要求：受保护接口

#### `switchRole(role)`

- 用途：切换角色
- 参数：`tenant` / `landlord`
- 要求：受保护接口

#### `deleteAccount()`

- 用途：注销账号
- 要求：受保护接口
- 效果：禁用用户、禁用身份映射、撤销现有 session

- 用途：地址转经纬度

### 3.8 `bootstrap.service.js`

#### `initCollections()`

- 用途：初始化必要集合

#### `initRegions()`

- 用途：初始化区域数据

#### `initAll()`

- 用途：同时初始化集合与区域

#### `cleanupTestUsers(phones?)`

- 用途：清理开发环境预留测试手机号对应的 `disabled` 历史账号
- 默认手机号：`13387395714`、`17364071058`
- 行为：仅删除 `disabled` 的历史 `users`，以及对应旧 `user_identities` / `user_sessions`

说明：

- 仅开发环境允许执行
- 页面层不得直接调用该能力

## 5. 错误码建议

| code | 含义 |
| --- | --- |
| `0` | 成功 |
| `400` | 参数错误 |
| `401` | 未登录或登录已过期 |
| `403` | 无权限 |
| `404` | 数据不存在 |
| `409` | 数据冲突 |
| `500` | 服务端异常 |

## 6. 接口约束

- 所有接口都必须返回统一格式
- 所有受保护接口必须校验 `auth.accessToken`
- token 无效、过期、被撤销时统一返回 `401`
- 所有写接口必须校验参数合法性
- 页面不允许拼接云函数名称和 action
- 前端登录成功后必须保存 `accessToken + userInfo`
- 页面刷新当前用户时应优先走 `userStore.refreshCurrentUser()`
