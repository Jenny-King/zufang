## 3. 常用联调命令

```bash
npm run deploy:cf -- -FunctionName house
```

```bash
npm run deploy:cf -- -FunctionName support,user,auth
```

```bash
npm run seed:houses6
```

```bash
node scripts/cleanup-test-users.js
```

说明：

- `deploy:cf` 用于按云函数名定向部署
- `seed:houses6` 会生成 6 条带图片、定位和城市区域字段的房源测试数据
- `cleanup-test-users.js` 用于清理开发环境预留手机号对应的禁用历史账号

## 4. 手动验收清单

### 4.1 登录注册

- 微信登录成功，首页正常打开
- 验证码登录成功，`accessToken + userInfo` 正常写入本地
- 密码登录成功
- 注册成功后自动登录
- 重置密码后可使用新密码重新登录

### 4.2 找房首页

- 首页城市定位正常显示
- 城市切换后，区域筛选与列表同步刷新
- 关键词搜索、区域筛选、户型筛选、租金筛选可用
- 精选推荐和最新发布列表正常展示
- 点击房源卡片可进入详情页

### 4.3 房源发布与管理

- 房东可从“发布”Tab 进入房源管理页
- “发布新房源”能进入三步发布流程
- 发布成功后返回房源管理页并自动刷新
- 编辑房源会复用发布流程并正确回填数据
- 下架 / 删除操作立即反映到列表中

### 4.4 收藏、历史与通知

- 收藏房源后，“我的收藏”列表正常出现
- 浏览房源后，“浏览历史”列表正常出现
- 通知列表只展示系统 / 业务通知，不展示聊天类通知
- 反馈提交成功后会新增一条系统通知

### 4.5 聊天与客服

- 从房源详情发起聊天可创建或进入会话
- 会话列表和聊天详情均可正常收发消息
- 客服中心可进入在线客服
- 问题反馈可提交成功

### 4.6 我的 / 设置

- “我的”页头像、昵称、角色、实名状态、微信状态显示正确
- 头部三项统计：收藏 / 历史 / 通知 数量正确
- 设置页：换绑手机号、修改密码、绑定微信、绑定邮箱、身份资料、注销账号可用
- 多账号缓存切换正常
- 删除缓存账号记录不会误删云端账号

## 5. 答辩演示建议顺序

1. 使用租客账号登录，展示首页找房、筛选和房源详情
2. 从详情页完成收藏和联系房东
3. 进入“我的”，展示收藏、历史、通知和设置入口
4. 切换到房东账号，展示角色切换效果
5. 进入“发布”Tab，完成发布、编辑或下架演示
6. 打开客服中心，展示在线客服或反馈提交

## 6. 当前联调建议

- 优先保证 `auth / user / house / support` 四个云函数是最新部署版本
- 清空旧房源数据后，再执行 `npm run seed:houses6`
- 如需重新初始化区域数据，先确认 `bootstrap` 已显式标记为非生产环境
